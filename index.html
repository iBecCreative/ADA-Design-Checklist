ADA Design Checklist

Color

  • Sufficient Color Contrast
      Sufficient color contrast is provided between foreground and background colors, verified using color contrast checkers such as the Stark plugin for Sketch,Adobe XD & Figma, Cluse plugin for Sketch, or WebAim’s online Contrast Checker. This includes text overlaying images, background gradients, buttons and other elements. Does not apply for logos or incidental text (for example, text on a sign in a photograph). Learn more about color contrast.

      Contrast Ratios:

      • Normal text (text is less than 18 point if not bold and less than 14 point if bold):
        AA (4.5:1) / AAA (7:1)
      • Large text (text is at least 18 point if not bold and at least 14 point if bold):
        AA (3:1) / AAA (3:1)
  • Proper Use of Color
      Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. Additional indicators that do not rely on color perception are provided, such as an asterisk in addition to color to indicate required form fields, an underline in addition to color to indicate a link, and labels or patterns on graphs. Learn more about accessible use of color.

Content

  • Headings & Spacing to Group Content
      Headings are styled and used in order to group content and improve readability. Content is easy to scan and understand. Whitespace and proximity are used appropriately, making relationships between content more apparent. Learn more about the use of headings and spacing.
  • Viewport Sizes
      Content is readable and displays in the correct order when presented in different sized viewports. This has been verified by viewing the page at a minimum viewport width of 320px.
  • Image & Media Alternatives
      Design includes image and media alternatives such as audio transcriptions, audio descriptions of video, icon labels, and captions and descriptions for tables or complex graphs. Learn more about including images and media alternatives in your design.
  • Automatic Content Controls
      Visible controls are provided for users to stop/play auto-playing image slideshows, video, sound and animations. This applies to carousels, images sliders, embedded video, background audio, GIFs, and auto-playing CSS animations. Learn more about providing controls for auto-playing content.
  • GIFs
      GIF loops stop after 5 seconds. Any GIFs that contain flashing or rapid transitions between light and dark colors are likely to cause seizures and should not be used. The Trace Center's Photosensitive Epilepsy Analysis Tool (PEAT) is a free, downloadable resource for developers to identify seizure risks in their web content and software.
  • Dashes
      Dashes are avoided to improve the user experience for those using screenreader technologies (example: instead of 5–7, write 5 to 7).
  • Images Optimized
      All images are compressed using either web processors such as Optimizilla.com and Compressor.io, or desktop apps such as FileOptimizer and ImageOptim. Images are generatedat a max of @2x for retina displays or exact image size when necessary instead of resizing it with CSS/HTML as this can be a heavy process for the browser to perform.

Interactions

  • Identifiable Interactive Elements
      Distinct, easy to identify styles are assigned to interactive elements such as links and buttons. Interactive elements are styled consistently throughout. Mouse hover causes the appearance of interactive elements to change in two or more ways. Keyboard focus and touch-screen activation states cause the appearance to change in one or more ways (browsers have native states for these, but it’s possible that they will need to be customized in order to provide sufficient color contrast). Learn more about accessible interactive elements.
  • Clear, Consistent Navigation
      Website page naming structure is consistent and the navigation has consistent styling and positioning. More than one method of navigating the website is provided, such as through a site search feature or a site map page. Users are able to understand where they are in the website by using orientation cues such as breadcrumbs, headings, or active states on navigation links. Learn more about designing an accessible navigation.
  • Descriptive Link Titles
      Link titles are descriptive and easy to understand out of context given that screen readers often navigation from link to link, skipping the surrounding text. Short, non-descriptive link text such as “click here” or “learn more” are not used. When the link points to a PDF or popup, it is indicated at the end of the link (example: “Sign up for our newsletter (opens popup)” or “View our annual report (opens PDF in a new window)”. Learn more about writing descriptive link text.
  • Form Labels
      All form fields have descriptive labels adjacent to the field. For left-to-right languages, labels are positioned to the left of the field or left-aligned above the field. Instructions and examples are provided when necessary to provide direction or require specific formatting. Learn more about designing accessible web forms.
  • Identifiable Feedback
      Feedback is provided for user interactions such as a form submission confirmation, error, or notification that something has changed on the page. Messages providing confirmations, instructions, and error information are easy to understand and identify. Important feedback that requires the user to take action is styled prominently. Learn more about providing easily identifiable feedback.
